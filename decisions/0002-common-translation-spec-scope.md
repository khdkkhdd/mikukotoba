# 공통 번역 기능 명세를 사이트별 명세와 분리하여 작성

Status: accepted
Date: 2026-02-19

## Context

기존 계획에서는 유튜브·트위터·웹페이지 3개 사이트별 명세만 예정되어 있었으나, 번역 파이프라인(엔진 선택, 표시 방식, 설정, 용어집 등)이 모든 사이트에서 공유되고 있어 중복 기술이 불가피했다. 또한 초기 작성 시 복잡도 알고리즘, 캐시 아키텍처, AI 프롬프트 규칙 등 기술적 구현 세부사항이 포함되어 기능 명세 원칙(decision 0001)에 어긋났다.

## Decision

`docs/translation_feature.md`로 공통 번역 기능 명세를 별도 문서로 작성한다. 사이트별 명세(유튜브, 트위터, 웹페이지)는 각 사이트 고유의 동작만 기술하고, 공통 로직은 이 문서를 참조한다. 기술적 세부사항(알고리즘, 내부 구조, 프롬프트 등)은 철저히 배제하고, 사용자가 보고 조작하는 것만 기술한다.

## Consequences

### Positive
- 사이트별 명세에서 번역 엔진, 설정, 용어집 등을 반복 설명할 필요가 없음
- 공통 기능 변경 시 한 문서만 수정하면 됨
- 기능 명세 원칙(사용자 관점)이 명확해짐

### Negative
- 원래 계획의 4개 문서에서 5개 문서로 증가
- 사이트별 명세를 읽을 때 공통 문서도 함께 참조해야 전체 그림이 보임

## Alternatives Considered

- **사이트별 명세에 공통 내용 중복 기술**: 동일한 설명이 3개 문서에 반복됨. 유지보수 시 불일치 위험. Rejected.
- **기술적 세부사항 포함**: 리팩토링 시 구현이 바뀌므로 금방 무의미해짐 (decision 0001과 동일한 논리). Rejected.
